# Contributing / Publishing

## Publishing to the Zed extension store

Zed extensions **do not** support an author self-publish command — every extension is reviewed and distributed through the official central repo. Flow:

1. Fork [zed-industries/extensions](https://github.com/zed-industries/extensions).
2. In your fork, add this repo as a submodule (use the **latest commit on the `main` branch**, not a tag/detached commit):

   ```sh
   git submodule add https://github.com/Nahida-aa/hover-dict.git extensions/hover-dict
   git add extensions/hover-dict
   ```

3. Append to the top-level `extensions.toml` (version must match `extension.toml`):

   ```toml
   [hover-dict]
   submodule = "extensions/hover-dict"
   version = "0.1.0"
   ```

4. Run `pnpm sort-extensions`, then open a PR to `zed-industries/extensions`.
5. After merge, the official pipeline packages `extension.wasm` and publishes it to the Zed store.

### Zed audit hard constraints (all satisfied)

- Extension ID/name must not contain `zed`/`extension` → we use `hover-dict` ✅
- The language server must **not** be bundled with the extension; it must be downloaded at runtime → `src/lib.rs`'s `download_ls` pulls `hover-dict-ls-<version>-<target>.zip` from the GitHub release ✅
- The repo root must have an accepted LICENSE (MIT) ✅
- The submodule must use an HTTPS URL and point at a branch commit

### How the LS binary gets built

Tagging `v*` triggers `.github/workflows/release.yml` (generated by cargo-dist), which compiles `hover-dict-ls` for 6 platforms and uploads `hover-dict-ls-<ver>-<triple>.zip` to the matching GitHub release. Asset names have **no `v` prefix**, aligned with `download_ls`.

> Build note: `build.rs` embeds the dictionary via `include_dir!` and must **not** `canonicalize()` the dict path — on Windows that produces a `\\?\` UNC prefix that breaks `include_dir!`.

## Dictionary data

The built-in ~760k-word English dictionary is derived from **[ECDICT](https://github.com/skywind3000/ECDICT)** (skywind3000), fetched from its `ecdict.csv` and split into per-prefix JSON files under `dict/`. Word fields: `w` = word, `p` = phonetic, `t` = translation.

- **Dictionary data license**: ECDICT is released under **[CC BY-NC 4.0](https://creativecommons.org/licenses/by-nc/4.0/)** (Attribution-NonCommercial). The extension is free and open-source and does not commercialize the dictionary data.
- **Extension code license**: MIT (see `LICENSE`).

If you rebuild/refresh the dictionary, regenerate `dict/*.json` from the ECDICT source and keep this attribution intact.
